# AI-Driven Operational Incident Intelligence Pipeline

## 📌 Project Overview
This project demonstrates an end-to-end data engineering and analytics pipeline designed to optimize IT incident management. It integrates **Python**, **SQL**, and **Generative AI (Gemma 3)** to transform raw operational data into strategic business insights.

As an Industrial Engineer focused on process improvement, I developed this tool to bridge the gap between technical failures and executive decision-making.

## 🚀 Key Features
- **Professional Data Generation:** Custom Python scripts to simulate 1,000+ realistic IT incidents with logical technical contexts.
- **Robust ETL Pipeline:** Automated extraction, transformation, and loading (ETL) from CSV to a structured SQLite database.
- **AI Orchestration:** Integration with **Google Gemini (Gemma 3)** to perform automated root cause analysis and generate process improvement recommendations.
- **Interactive Analytics:** A high-end Power BI Dashboard focused on Operational Risk and Efficiency KPIs.

## 🛠️ Tech Stack
- **Language:** Python 3.x
- **Database:** SQLite
- **AI Model:** Google Gemma 3 (via Google Generative AI API)
- **Data Analysis:** Pandas
- **Visualization:** Power BI (DAX)

## 📁 Project Structure
- `Data/`: Raw CSVs and processed SQLite database.
- `Source/Python/`: 
    - `Incident_data_generator.py`: Script for professional data seeding.
    - `data_ingestion_sql.py`: ETL logic.
    - `incident_orchestrator.py`: AI analysis engine.
    - `export_final_report.py`: Traceability and export script.
- `Outputs/`: Final analyzed report for Power BI consumption.

## 📊 Business Impact
By automating incident classification and suggesting systemic fixes, this pipeline reduces the manual workload of Business Analysts and identifies recurring bottlenecks in the supply chain and IT infrastructure, driving continuous innovation.
